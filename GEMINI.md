# アプリケーション仕様
SNS営業窓口 AIエージェント」「Phronel」ディレクトリ先：./phronel_ai_agent/
- 次のようなAIエージェントアプリケーションを開発します。
 - このAIエージェントは、管理者が提供した情報に基づいて、SNS(X)で情報を展開営業活動をします。
 - SNSにおいて情報を配信したりSNSユーザに対して返信したり、また、自分の活動の状況を管理者に報告したりします。

# アプリケーション機能概略
- 機能1: X のアカウント連携設定
 - 管理者から、営業活動するためのドキュメントやWebURL情報を教えてもらい、Xに登録済みのメールアドレスとAPIをもらう。
 - もらったメールアドレスと素材情報から情報を整理しAPI上のXアカウント

- 機能2: X アカウントで営業活動配信/ 返信活動
 - 定期的に営業活動するためのドキュメントWebURL情報の関する広告などのつぶやきを、Xから配信します
 - 配信に対してXの別のユーザからのレスポンスに対し、リプライ・リポスト・フォローなどを行います

- 機能3: 営業活動結果の報告
 - 機能2で活動した結果のサマリーを管理者に定期的に報告します

# 製品戦略
既存のツールが「作業の効率化」に主眼を置いているのに対し、Phronelは**「営業ペルソナを持つ自律型AIエージェントによる、高品質な関係構築の自動化」**を目指します。以下に具体的な差別化戦略を提案します。

- 戦略1: 人間らしい「ペルソナ」による高品質な関係構築
  管理者が設定した情報に基づき、特定の営業担当者のような一貫した性格、口調、専門性を持つ「ペルソナ」をAIに持たせます。これにより、自動化された応答であっても人間味のある、質の高いインタラクションを実現し、ユーザーとの長期的な関係構築に貢献します。

- 戦略2: 目標達成に向けたプロアクティブ・自律的なアクション
  単に投稿や返信を行うだけでなく、管理者が設定したゴール（例: 月次フォロワー100人増、Webサイトへの誘導50件）に基づき、AIが自律的に戦略を考え、実行します。
具体的には、関連性の高いキーワードを発しているユーザーを自ら探し出して「いいね」やフォロー、リプライを行うなど、プロアクティブ（能動的）な営業活動を展開します。AI自身がPDCAサイクルを回し、戦略を最適化し続けます。

- 戦略3: AIの"思考"を可視化するインサイト重視のレポーティング
  活動結果の数値サマリーだけでなく、「なぜこの投稿がエンゲージメントを高めたのか」「どのようなユーザー層にアプローチした結果、目標達成に近づいたのか」といった、AIの判断根拠やインサイトを管理者に報告します。これにより、管理者はAIの活動を理解し、次の戦略立案に役立つ知見を得ることができます。

# アーキテクチャ構成
アプリケーションはWin常駐できるOSSアプリ（将来的にはGUI化も視野）とし、まずはPythonを利用した高度なCLI/CUIインターフェースを持つ軽量アプリケーションとして開発する。

- **CUI/CLI フレームワーク**: 
 - **Textual**: ターミナル上でリッチなUI（ボタン、スクロール、非同期更新）を実現するTUIフレームワーク。AIエージェントの常駐監視・操作盤として利用。
 - **Typer**: コマンドライン引数解析。起動、設定、メンテナンスなどのCLIコマンドを直感的に提供。
 - **Rich**: ログの整形、Markdown表示、プログレスバーなどのリッチな出力。

- **コアロジック**: Python
 - 非同期（async/await）を基本とし、UIをブロックせずにAI処理やAPI通信を実行。
 - AIエージェントのコアとなるロジック（LLM API連携、 X API連携、 タスクスケジューリングなど）を、requestsやtweepy、httpxなどのライブラリを利用して実装。
 
- AIコアエンジン: DSPy Framework
 - DSPyを中核フレームワークとして採用し、プロンプトの自動最適化と高品質な応答生成を目指します。
 - **プロンプト最適化**: DSPyの`Signature`と`Optimizer`を活用し、SNSへの返信のようなタスクにおいて、データに基づき最も効果的なプロンプトを自動的に構築・改善します。これにより、手動でのプロンプトエンジニアリングの負荷を軽減し、継続的な品質向上を実現します。
 - **LLM API (Gemini, OpenAI 등)**: DSPyモジュール内で各種LLMを切り替え可能にし、柔軟なモデル選択をサポートします。
 - **RAG (Retrieval-Augmented Generation)**: 提供されたドキュメントやURLを知識ベースとし、DSPyの`Retrieve`モジュールと連携して、文脈に基づいた正確な情報を提供します。VectorDatabaseにはChromaDBなど、Pythonと親和性の高いツールを利用します。

- データストレージ: SQLite
 - ユーザーのPC内に設定やデータを保存するローカルデータベースとして最適です。Python標準のsqlite3モジュールで簡単に扱えます。
